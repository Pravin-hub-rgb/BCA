Indentation in Python

  • Indentation is a fundamental aspect of defining the programming style in Python.
  • Spaces (or tabs) are used for indentation, helping to visually organize code and indicating the hierarchy of statements within functions or classes.
  • Python strictly follows significant indentation, so unnecessary indentation or mixing spaces and tabs can lead to errors. Even a single space can result in an indentation error.

Rules of Indentation:

  • Indentation is crucial for demarcating the beginning and end of code blocks.
  • Consistency in the use of spaces or tabs for indentation is important throughout the codebase.
  • The conventional practice in Python is to use four spaces for each level of indentation.
  • Mixing tabs and spaces is discouraged, as it can cause indentation-related issues.
  • Improper indentation can lead to syntax errors or unintended logical structures in the program.
  • Mandatory indentation is required after certain statements, such as loops, conditionals, function definitions, and class definitions.

It's essential to adhere to these rules for maintaining readable and error-free Python code.

// & ** Operator in Python

The // and ** operators are special operators in Python that serve different purposes.

  • // Operator: The double-slash (//) operator is the floor division operator. It performs division and returns the integer part of the result, discarding any fractional remainder.
  • ** Operator: The double-asterisk (**) operator is the exponentiation operator. It raises the left operand to the power of the right operand. For example, 2**3 equals 8 (2 raised to the power of 3).

Examples of using these operators:

                
# Floor division (//) example
result_floor_division = 10 // 3 # Result: 3

# Exponentiation (**) example
result_exponentiation = 2 ** 4 # Result: 16
